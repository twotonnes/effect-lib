3.1 Basic Monadic FunctionsðŸ”—â„¹

procedure

(return v) → (Eff A)

  v : A
Wraps a pure value into the Eff monad.

This is the return or pure operation of the monad. It represents a computation that has no effects and immediately produces the given value.

Examples:
> (return 42)

- : (U (effect Positive-Byte) (pure Positive-Byte))

(pure 42)

> (return "hello")

- : (U (effect String) (pure String))

(pure "hello")

procedure

(>>= m f) → (Eff B)

  m : (Eff A)
  f : (-> A (Eff B))
Monadic bind operation (pronounced "bind"). Sequences two effectful computations together.

  • If m is a pure value, f is applied to that value, yielding the next computation.

  • If m is an effect, a new effect is created with the same description but a new continuation that threads the result through f.

This operation is the key to composing effectful computations while preserving their structure.

Example:
> (>>= (return 1)
       (lambda (x) (return (+ x 1))))

eval:4:0: Type Checker: type mismatch

  expected: Number

  given: Any

  in: 1

procedure

(bind m f) → (Eff B)

  m : (Eff A)
  f : (-> A (Eff B))
An alias for >>=. This is the prefix form of the bind operation.

3.2 Effect InterpretationðŸ”—â„¹

procedure

(run m handle) → Any

  m : (Eff Any)
  handle : (-> (effect Any) (Eff Any))
Interprets an effectful computation by repeatedly applying a handler until a pure value is obtained.

The handle function is called each time an effect is encountered. It receives the effect structure and should return another Eff computation—either a pure result, or another effect to be handled.

  • If m is already pure, its value is immediately returned.

  • If m is an effect, handle is applied to it, and the result is recursively run.

Example:
> (run (return 42)
       (lambda (eff) (error "No effect should occur")))

- : Any

42
