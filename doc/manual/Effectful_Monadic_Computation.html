1 Effectful Monadic ComputationðŸ”—â„¹

The effect-lib library provides a typed, extensible effects monad for Racket. This allows you to separate the description of side effects from their execution, enabling flexible effect handling strategies.

1.1 Core TypesðŸ”—â„¹

struct

(struct pure A (value))

  value : A
Represents a pure value in the effects monad. This is one of the two constructors for Eff.

struct

(struct effect A (description k))

  description : effect-desc
  k : (-> Any (Eff A))
Represents an effect waiting to be handled. The description identifies the type of effect, and k is a continuation that will receive the effect’s result and return another Eff computation.

struct

(struct effect-desc ()
    #:extra-constructor-name make-effect-desc)
An extensible descriptor for effects. This base struct can be extended by subclasses to hold metadata about specific effect types.

type

Eff : (All (A) (U (pure A) (effect A)))

The Eff monad type. An Eff computation is either a pure value or an effect waiting to be handled.

1.2 Monadic OperationsðŸ”—â„¹

procedure

(return v) → (Eff A)

  v : A
Wraps a value into the Eff monad. Equivalent to pure.

procedure

(>>= m f) → (Eff B)

  m : (Eff A)
  f : (-> A (Eff B))
Monadic bind operation. Sequences two effectful computations: if m is pure, applies f to its value; if m is an effect, creates a new effect with the same description but a continuation that threads the result through f.

procedure

(run m handle) → Any

  m : (Eff Any)
  handle : (-> (effect Any) (Eff Any))
Interprets an effectful computation by repeatedly applying the handle function to effects until a pure value emerges. The handler receives an effect structure and should return another Eff computation, allowing it to either produce a result or defer the effect.

1.3 Syntactic SugarðŸ”—â„¹

syntax

(do stmt ...)

 
stmt = [var <- expr]
  | [var : type <- expr]
  | expr
Syntactic sugar for monadic sequencing using do-notation. The do form sequences multiple statements and a final expression:

  • [var <- expr] binds the result of an effectful computation to var for use in subsequent statements.

  • [var : type <- expr] is the same as above but with an explicit type annotation for var.

  • expr (a plain expression) evaluates the expression and discards its result, continuing with the next statement.

All sequences expand to nested >>= calls, threading results through.

Example:
> (do [x : Integer <- (return 1)]
      [y : Integer <- (return 2)]
      (return (+ x y)))

- : (U (effect Integer) (pure Integer))

(pure 3)

syntax

(with-effect-handlers (clause ...) expr ...)

 
clause = [pattern body ... (abort result)]
  | [pattern body ...]
Macro for pattern-matching effect handlers. Each clause matches on an effect structure and either:

  • Uses (abort result) to short-circuit and return a pure result without invoking the continuation.

  • Returns an expression whose result is passed to the effect’s continuation via >>=.

The handler is applied to all effects produced by expr ... until a pure value is obtained.

Example:
> (with-effect-handlers ([(id-effect v) (return v)])
      (do
          [x <- (id "some value")]
          (return x)))

- : Any

"some value"
