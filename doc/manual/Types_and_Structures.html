2.1 Core TypeðŸ”—â„¹

type

Eff : (All (A) (U (pure A) (effect A)))

The Eff monad type. An Eff computation is either a pure value (containing a result with no effects) or an effect structure (representing an effect waiting to be handled).

2.2 Pure ValuesðŸ”—â„¹

struct

(struct pure A (value)
    #:transparent)
  value : A
Represents a pure value in the effects monad—a computation that has no effects and directly produces a result.

This is one of the two constructors for the Eff type.

2.3 EffectsðŸ”—â„¹

struct

(struct effect A (description k)
    #:transparent)
  description : effect-desc
  k : (-> Any (Eff A))
Represents an effect waiting to be handled. When an effect handler processes this structure:

  • description is an effect-desc that identifies and describes the type of effect. Custom effect types should extend this struct.

  • k is a continuation (a function) that will receive the effect’s result and return another Eff computation. This allows sequencing computations.

This is the other constructor for the Eff type.

struct

(struct effect-desc ()
    #:extra-constructor-name make-effect-desc
    #:transparent)
The base descriptor for effects. This is an extensible struct that serves as the parent type for specific effect descriptors. Subclass this struct to represent particular types of effects.
