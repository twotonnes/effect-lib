An effect monad is a structure that represents computations that may produce effects. Rather than executing effects immediately, the effect monad describes the effects and provides a mechanism to interpret them later.

The core idea is that an effectful computation is either:

  • A pure value with no effects, or

  • An effect waiting to be handled.

This structure makes it easy to:

  • Compose effectful operations cleanly using monadic sequencing,

  • Handle different effect types with different strategies,

  • Test code by substituting mock effect handlers, and

  • Reason about what effects a computation may perform.
